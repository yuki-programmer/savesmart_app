# ダークパターン

ユーザーを欺く・操作する非倫理的な手法。避けるべきパターン。

## なぜ避けるべきか

1. **法的リスク**: GDPR、消費者保護法違反の可能性
2. **信頼喪失**: 発覚時のブランドダメージ
3. **長期的損失**: 短期CVR向上より長期LTV低下
4. **アプリストア規約違反**: リジェクトリスク

## 主要なダークパターン

### Roach Motel（ゴキブリホイホイ）
入るのは簡単、出るのは困難。

**例**: ワンクリック登録、解約は電話のみ
**対策**: 登録と同じ導線で解約可能に

### Trick Questions（引っかけ質問）
紛らわしい文言で意図しない選択をさせる。

**例**: 「メールを受け取りたくない場合はチェック」（二重否定）
**対策**: 明確でシンプルな文言

### Privacy Zuckering
意図せず個人情報を公開させる。

**例**: デフォルトで全公開、複雑なプライバシー設定
**対策**: プライバシー設定を分かりやすく、デフォルトは非公開寄りに

### Misdirection（誘導）
視覚的に別の選択肢へ誘導。

**例**: 「はい」ボタンだけ目立たせる、小さな「スキップ」
**対策**: 選択肢を公平に表示

### Hidden Costs（隠れコスト）
最後に手数料・送料を追加。

**例**: カート投入後に送料¥2,000
**対策**: 総額を早めに明示

### Bait and Switch（おとり商法）
期待と異なる結果。

**例**: 無料と見せて実は条件付き
**対策**: 条件を最初から明示

### Confirmshaming（確認で恥をかかせる）
断る選択肢を恥ずかしい文言に。

**例**: 「いいえ、お金を節約したくありません」
**対策**: ニュートラルな文言「今はスキップ」

### Disguised Ads（偽装広告）
コンテンツに見せかけた広告。

**例**: 記事風の広告
**対策**: 「広告」「PR」と明示

### Forced Continuity（強制継続）
トライアル終了後の自動課金。

**例**: 無料期間終了を分かりにくく
**対策**: 終了前の明確な通知、簡単なキャンセル

### Friend Spam（友人スパム）
連絡先に勝手に招待を送信。

**例**: 連絡先アクセス許可で自動招待
**対策**: 招待は個別に明示的な同意を得て

## グレーゾーンの判断基準

以下の質問に「いいえ」なら再考：

1. **透明性**: ユーザーは何が起きているか理解できるか？
2. **可逆性**: ユーザーは簡単に取り消せるか？
3. **公平性**: 両方の選択肢が公平に提示されているか？
4. **誠実性**: 説明に誇張や省略はないか？
5. **自律性**: ユーザーの意思決定を尊重しているか？

## 倫理的な代替手段

| ダークパターン | 倫理的代替 |
|--------------|----------|
| 解約困難 | 簡単解約＋理由ヒアリング＋再開特典 |
| 隠れコスト | 総額表示＋送料無料の閾値表示 |
| デフォルトON | 明確な説明付きオプトイン |
| 希少性の偽装 | 本当の在庫数表示 |
| 引っかけ文言 | シンプルで明確な文言 |
